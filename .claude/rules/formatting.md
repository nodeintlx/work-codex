# Formatting Rules

## Date & Time
- All dates and times in Eastern Time (ET)
- Date format: YYYY-MM-DD for data files, "Day, Month Date, Year" for human-readable output
- Time format: 12-hour with AM/PM for briefings, 24-hour for data files
- Always specify timezone when ambiguity is possible

## Output Style
- Use markdown formatting for all documents and reports
- Use YAML for structured data files (goals, tasks, schedules)
- Use JSON for contact records and machine-readable data
- Be concise — lead with conclusions, support with evidence
- Use bullet points over paragraphs
- Use tables for comparative data

## Documents
- File names: lowercase, hyphenated (e.g., meeting-brief-2026-02-22.md)
- Include date prefix for time-sensitive documents
- Always include a clear title/header

## Communications
- Match tone to context (see company-specific CLAUDE.md files)
- Keep emails concise — busy executives respect brevity
- Include clear call-to-action when a response is needed
- Draft subject lines that are specific and scannable
